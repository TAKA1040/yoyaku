【前提】
- 本プロジェクトは「Supabase Auth 方式」。NextAuth は使用しません。
- すでに初期設定（ドメイン、Supabase、GCP OAuth、Vercel 環境変数、ローカル .env.local）が完了しています。
- 使用サービス：Supabase / Vercel / GitHub / Google Cloud Console。
- 重要ファイル：.gitignore は漏洩防止設定済み、.env.local は存在（公開系のみ）。

【環境の要点】
- NEXT_PUBLIC_SUPABASE_URL=（設定済）
- NEXT_PUBLIC_SUPABASE_ANON_KEY=（設定済）
- SUPABASE_SERVICE_ROLE_KEY=（Vercel のみで保持。ローカルやフロントで使用禁止）
- Supabase Auth：Site URL と Redirect 設定済（https://<project-ref>.supabase.co/auth/v1/callback）。

【ルール】
1) .env.local を直接編集・上書きしない。必要なら生成プロンプト手順を使う。
2) NEXTAUTH_* を提案・追加しない。
3) 秘密値を画面に表示しない。必要ならファイル出力のみ。
4) 追加で必要な ENV があれば、理由と使用箇所を明記してから提案する。

【依頼】
- 上記前提を踏まえ、SPAベース（Next.js）で機能実装を進めてください。
- 生成するコードには、どのファイルに作るかの**パス**と**目的**を先頭に明記してください（例：`/src/app/login/page.tsx`）。
- 依存追加が必要な場合は `package.json` 変更差分を提示し、理由を書いてください。
- 実装が完了したら、動作確認手順（コマンドと確認ポイント）を最後に出してください。
